Ocssipital

A grid creation boilerplate for SASS

Download Fork it

What and why

    Ocssipital is a lightweight and reusable CSS grid boilerplate that can be used in any project at any time.

    Compared to other systems, Ocssipital's one and only focus is providing a flexible grid that is deeply customizable enabling you to use your own classes and breakpoints.

    Built taking into consideration the W3C Grid Specification, Ocssipital's aim is to be prepared for the future.

Just grids

There a lot of frameworks that include multiple UI components and widgets, that turn out to be large and complex.

Ocssipital focuses on grids, and does not provide any extra code related to UI elements.

Built for designers, find in Ocssipital a great tool and partner to help you scale and adapt your designs for any device.

Native Flexbox support

By default, Ocssipital uses the recent Flexbox W3C specification for its grid. Check its browser support here.

If flexbox is not the right choice for you right now, don't worry, Ocssipital already includes a traditional float layout that you can use as well.

Usage


<!-- Create a container with children columns -->
<div class="grid-container">
  <div class="grid-column-1">Three</div>
  <div class="grid-column-2">Two</div>
  <div class="grid-column-3">Three</div>
</div>


<!-- Change column sizes for specific viewports -->
<div class="grid-container">
  <div class="grid-column-12-s"></div>
  <div class="grid-column-6-m"></div>
  <div class="grid-column-2-l"></div>
</div>


<!-- Create grid gaps or hide elements -->
<div class="grid-container">
  <div class="grid-gap-left-2"></div>
  <div class="grid-gap-right-2"></div>
  <div class="visible-xl"></div>
</div>

Settings

  • Flexbox (Boolean)

    Defines whether to use flexbox layout or traditional floats. By default: true.

    
$flexbox:true;

  • Column Count (Int)

    Defines the amount of maximum columns for the grid. By default: 12.

    
$column_count:12;

  • Column Prefix (String)

    Defines the class prefix that will be applied to each column. By default: grid-column.

    
$column_prefix:"grid-column";

  • Column Container (String)

    Defines the name of the grid container. By default: grid-container.

    
$container_name:"grid-container";

  • Viewports (Array)

    Defines your website viewports and breakpoints. By default: s, m, l, xl.

    
$viewports:(s, 0, 639px, true) …;

  • Debugging (Boolean)

    Highlights all columns to be able to debug the layout. By default: false.

    
$debug:false;

  • Grid Gaps (Boolean)

    Enables gaps as empty placeholders to use in your layout. By default: true.

    
$gaps:true;

  • Grid Gap Prefix (String)

    Defines the class prefix related to the grid gaps. By default: grid-gap.

    
$gap_prefix:"grid-gap";

  • Grid Gutter Unit (String)

    Defines the measuring unit for the gutter (atm. only px). By default: px.

    
$grid_gutter_unit:"px";

  • Grid Gutter Size (Int)

    Defines the size of the gap for the current selected unit. By default: 0.

    
$grid_gutter_size:0;

Mixins

  • Mediaqueries

    Use this mixin to include custom styles in a specific viewport.

    
@include mediaquery($viewport){}

  • Change Column

    Changes an element's current column size for a different one.

    
@include change_column($number){}
